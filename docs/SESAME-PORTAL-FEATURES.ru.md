# SesamePortal: описание возможностей

## Назначение

SesamePortal - локально устанавливаемый портал видеонаблюдения для работы с
одним или несколькими серверами SesameDVR. Он объединяет управление
пользователями, группами доступа, камерами, картой объектов и авторизацией
просмотра видео в едином web-интерфейсе.

Портал может использоваться для частных, корпоративных и распределённых систем
видеонаблюдения, где важно централизованно управлять доступом к камерам, но при
этом хранить и обрабатывать видео локально на собственных серверах.

## Ключевые возможности

- управление несколькими серверами SesameDVR из одного интерфейса;
- добавление и настройка камер с привязкой к DVR-серверу;
- два режима работы камеры: полное управление через DVR или read-only доступ к
  уже существующему потоку;
- разграничение доступа пользователей через группы;
- мозаика камер с preview и переходом в SesameDVR embed player;
- просмотр камер на карте OpenStreetMap;
- отображение положения, направления и угла обзора камер на карте;
- автоматический подбор масштаба и кластеризация камер на карте;
- поиск камер по названию в режимах мозаики и карты;
- пагинация мозаики и выбор плотности отображения от 2 до 6 камер в ряду;
- персональное избранное для каждого пользователя;
- ежедневные персональные playback-токены с перекрытием ротации;
- постоянные статические токены для интеграций;
- auth-backend для SesameDVR;
- локализация интерфейса на тот же набор языков, что и SesameDVR;
- мобильная версия интерфейса и удобный просмотр видео с телефона;
- отображение времени в часовом поясе браузера пользователя;
- журнал действий администратора;
- поиск и пагинация в административных списках;
- резервное копирование и восстановление данных через CLI;
- установщик для nginx, php-fpm, SQLite и Let's Encrypt SSL.

## Административный интерфейс

Администратор получает полный доступ к настройкам портала и может управлять
всеми основными сущностями системы.

### Серверы SesameDVR

В SesamePortal можно добавить несколько DVR-серверов. Для каждого сервера
задаются название, URL и management key. Портал умеет проверять доступность
серверов, сохранять результат последней проверки и показывать техническую
информацию о состоянии подключения.

Dashboard показывает версию SesameDVR, CPU, RAM, количество потоков и время
последней проверки. Если management token не указан, не расшифровывается или
DVR возвращает `HTTP 401`, портал объясняет причину вместо вывода сырого ответа
API.

Management key хранится в зашифрованном виде. Для ротации ключей шифрования
предусмотрена CLI-команда.

### Камеры

Для камер настраиваются:

- имя камеры;
- URL источника, например RTSP;
- сервер SesameDVR;
- ручной или автоматический выбор сервера;
- имя потока в SesameDVR;
- глубина архива;
- положение на карте;
- направление камеры;
- угол обзора;
- группы доступа;
- статус блокировки.

Камера может работать в одном из двух режимов:

- **Полное управление на DVR** - портал создаёт или обновляет поток на выбранном
  SesameDVR-сервере через management API.
- **Read-only** - портал не меняет конфигурацию DVR, а только использует уже
  существующий поток для preview, авторизации и playback.

Положение камеры можно редактировать прямо на карте. Направление обзора также
задаётся визуально, с подтверждением изменений для защиты от случайного
перетаскивания.

### Пользователи

Портал поддерживает пользователей с логином и паролем. Пароли хранятся в виде
hash, минимальная длина пароля - 6 символов.

Пользователь может быть:

- администратором с полным доступом;
- обычным пользователем только для просмотра разрешённых камер.

Администратор может создавать, редактировать, блокировать и удалять
пользователей, а также назначать им группы.

### Группы

Группы используются для разграничения доступа к камерам. В группу можно добавить
пользователей и камеры. Обычный пользователь видит только те камеры, которые
находятся в его доступных группах.

Администратор может создавать, изменять, блокировать и удалять группы. Если
группа заблокирована, камеры этой группы перестают быть доступными обычным
пользователям через портал и auth-backend.

Для больших инсталляций список групп выбирается через компактный список, а не
через длинную ленту кнопок. Это позволяет работать с большим количеством групп
без перегрузки интерфейса.

При редактировании группы пользователи и камеры выбираются через searchable
assignment picker: можно искать по названию, видеть только уже выбранные записи
и работать с сотнями пользователей или камер без длинного списка чекбоксов.

### Журнал действий

Административные операции фиксируются в журнале действий. Это помогает видеть,
кто и когда менял пользователей, группы, камеры и серверы.

Журнал поддерживает фильтрацию по действию, пользователю и тексту деталей.

## Пользовательский просмотр камер

Обычный пользователь работает только с теми камерами, к которым ему выдан
доступ через группы.

### Мозаика

Мозаика отображает доступные камеры в виде карточек. В карточке показываются:

- preview камеры;
- имя камеры;
- сервер или источник;
- кнопка открытия player;
- кнопка добавления или удаления из избранного.

Preview обновляется автоматически, чтобы пользователь видел актуальное состояние
камеры без ручного обновления страницы.

Карточки preview имеют фиксированное соотношение сторон `16:9`. При первичной
загрузке и обновлении preview показывается loader, а новая картинка подменяется
только после полной загрузки, чтобы пользователь не видел промежуточную
перерисовку изображения.

Мозаика поддерживает поиск по названию камеры, фильтрацию по всем доступным
камерам, избранному или выбранной группе, а также пагинацию для больших наборов
камер. Пользователь может выбрать плотность отображения: 2, 3, 4, 5 или 6 камер
в ряду. Переключение плотности выполняется на клиенте без повторной загрузки
страницы и без повторного запроса preview.

### Карта

Карта показывает доступные камеры на OpenStreetMap. Для камер отображаются:

- точка расположения;
- направление обзора;
- сектор обзора;
- preview в popup;
- кнопка добавления или удаления из избранного;
- ссылка на открытие видео.

Масштаб карты подбирается автоматически так, чтобы камеры выбранного фильтра
были видны на экране.

При уменьшении масштаба близко расположенные камеры объединяются в кластер с
числом камер. Это сохраняет читаемость карты на объектах с большим количеством
камер.

Карта поддерживает те же пользовательские фильтры, что и мозаика: все доступные
камеры, избранное, выбранная группа и поиск камеры по названию.

### Избранное

Каждый пользователь может добавить нужные камеры в личное избранное. Избранное
работает в мозаике и на карте.

Если пользователь потерял доступ к камере из-за изменения групп, блокировки
камеры или блокировки группы, такая камера перестаёт отображаться и в
избранном.

## Авторизация просмотра для SesameDVR

SesamePortal предоставляет auth-backend endpoint для серверов SesameDVR. DVR
может обращаться к порталу, чтобы проверить, имеет ли пользователь право
смотреть конкретный поток.

Для playback используются персональные daily-токены. Они меняются раз в сутки,
но имеют 6-часовое перекрытие: в первые часы новых суток валиден и новый, и
предыдущий токен. Это позволяет избежать обрыва доступа в момент ротации.

Для внешних интеграций можно создать дополнительный постоянный статический
токен конкретного пользователя.

## Установка и эксплуатация

SesamePortal устанавливается локально на сервер через install script.

Инсталлятор:

- устанавливает необходимые зависимости;
- разворачивает приложение в `/opt/sesame-portal/current`;
- создаёт state-директорию `/var/lib/sesame-portal`;
- настраивает nginx site для выбранного домена;
- подключает php-fpm;
- создаёт первого администратора;
- выпускает Let's Encrypt сертификат через `certbot certonly --webroot`;
- поддерживает повторный запуск в режиме repair/update;
- делает резервные копии важных файлов перед изменениями.

Инсталлятор создаёт отдельный nginx site-файл SesamePortal и не переписывает
чужие site-конфиги. Если установка или repair завершается ошибкой, он
откатывает release, config, nginx site, cron и SQLite-файлы к состоянию до
запуска.

После установки портал открывается по домену, переданному в `--domain`. Первый
администратор создаётся из параметров install script: `--admin-login` задаёт
логин, а `--admin-password` задаёт пароль. Если `--admin-login` не указан,
используется логин `admin`.

По умолчанию используется SQLite. Для более крупных инсталляций можно выбрать
MySQL или PostgreSQL через конфигурацию.

Интерфейс поддерживает языки SesameDVR: русский, английский, немецкий,
французский, испанский, итальянский, португальский, болгарский, польский,
китайский, японский, корейский, арабский и армянский. Язык выбирается в
выпадающем списке интерфейса.

## CLI-инструменты

Портал включает CLI-команды для обслуживания:

- миграции базы данных;
- создания администратора;
- ротации playback-токенов;
- ротации зашифрованных management secrets;
- резервного копирования;
- восстановления из backup.

## Для чего может использоваться SesamePortal

SesamePortal подходит для сценариев, где нужно:

- предоставить клиентам или сотрудникам доступ только к нужным камерам;
- объединить несколько SesameDVR-серверов в единый интерфейс;
- быстро находить камеры по объектам, группам или карте;
- дать пользователям удобный web-доступ к live/archive playback;
- хранить видео и управление внутри собственной инфраструктуры;
- подключить SesameDVR к внешней системе авторизации доступа;
- разделить права администраторов и обычных пользователей;
- обслуживать большое количество камер, пользователей и групп.

## Итог

SesamePortal дополняет SesameDVR уровнем централизованного управления доступом,
пользователями и объектами видеонаблюдения. SesameDVR отвечает за запись,
архивирование и проигрывание видео, а SesamePortal предоставляет удобный портал
для настройки прав, просмотра камер, работы с картой и интеграции с внешними
системами.
