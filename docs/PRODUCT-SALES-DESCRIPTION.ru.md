# Sesame DVR: описание продукта для продаж

Sesame DVR — это производительная серверная платформа для записи, хранения,
просмотра и публикации видео с IP-камер. Решение устанавливается на Linux-сервер,
подключается к RTSP/ONVIF-камерам, ведёт видеоархив, отдаёт live-видео и
предоставляет web-интерфейс для управления всей системой.

Продукт подходит для проектов, где важно быстро развернуть собственный DVR/NVR
сервер, контролировать хранение видео, предоставить доступ к live и архиву через
браузер или API и обслуживать большое количество потоков без тяжёлой VMS-
инфраструктуры.

## Ключевая ценность

Sesame DVR закрывает полный цикл работы с видеопотоками:

- подключение IP-камер и RTSP-источников;
- постоянная запись видеоархива;
- live-просмотр в браузере;
- быстрый поиск и просмотр архива по времени;
- выгрузка нужного фрагмента в MP4;
- масштабируемое хранилище архива в SingleVolume или MultiVolume режиме;
- публикация сервиса наружу через nginx и HTTPS;
- управление камерами, архивом, клиентами и диагностикой из единой web-панели;
- интеграция со сторонними системами через HTTP API и embed-плеер.

Для клиента это означает, что Sesame DVR можно использовать как самостоятельный
видеосервер, как backend для клиентского портала или как встраиваемый компонент
в отраслевую систему мониторинга.

## Возможности продукта

### Централизованная запись с IP-камер

Sesame DVR принимает видео от IP-камер и других источников, доступных по RTSP
или совместимым URL. Для каждой камеры можно настроить источник, имя, срок
хранения, режим авторизации, параметры ingest и состояние запуска.

Сервис автоматически запускает включённые потоки после старта и следит за тем,
что архив продолжает обновляться. Если поток завис или перестал писать новые
сегменты, система может перезапустить ingest без ручного вмешательства.

### Live-видео в браузере

Платформа отдаёт live-видео через HLS и native WebRTC/WHEP. Для H.264 потоков
WebRTC позволяет получить более интерактивный live-просмотр, а HLS остаётся
универсальным fallback-вариантом для широкого набора браузеров, плееров и
интеграций.

Для совместимых потоков поддерживается HEVC/H.265 WebRTC playback в браузерах,
которые умеют такой codec, а также G.711 audio passthrough без лишнего
транскодирования.

### Удобный видеоархив

Sesame DVR сохраняет архив на диске в self-initializing fMP4 сегментах,
индексирует их и позволяет быстро открывать нужный временной диапазон.
Пользователь может перейти к нужному моменту, посмотреть запись в браузере и
выгрузить выбранный участок в MP4.

Retention настраивается по камерам: старые записи очищаются автоматически, а
использование диска остаётся предсказуемым.

### Масштабируемое хранилище

Для стандартной установки достаточно одного archive root. Для крупных объектов
Sesame DVR поддерживает MultiVolume: архив можно распределять по нескольким
дискам, управлять записью на каждый volume, выводить volume из записи,
пересобирать catalog после переносов и сохранять доступ к остальным томам, если
один диск временно недоступен.

Публичная публикация через managed nginx отдаёт archive segments напрямую с
диска, включая MultiVolume URL `/dvr/v/<volume_id>/...`. Это снижает нагрузку
на приложение при активном просмотре архива.

### Web-панель администратора

Встроенный admin UI позволяет работать с системой без ручной правки файлов:

- добавлять, редактировать, запускать и останавливать камеры;
- включать и отключать потоки на постоянной основе;
- видеть, какие камеры сейчас пишут архив;
- смотреть live и archive playback;
- проверять codec, bitrate и ffprobe-информацию;
- видеть активных playback client users, WebRTC-клиентов и недавних HLS-клиентов;
- смотреть логи по каждому потоку и глобальный лог сервиса;
- контролировать CPU, память, диски, сеть и ingest-нагрузку;
- управлять томами хранения, audit и rebuild catalog;
- управлять ONVIF-устройствами и событиями движения.

Панель подходит и для первичной настройки, и для ежедневной эксплуатации.

### ONVIF и события движения

Sesame DVR умеет находить ONVIF-устройства, проверять их возможности и собирать
motion events через ONVIF PullPoint. События движения сохраняются отдельно от
видео и отображаются рядом с временной шкалой архива.

Это помогает быстрее находить важные моменты: оператор видит, где в архиве есть
движение, и может сразу открыть соответствующий фрагмент видео.

### Встраивание и интеграция

Продукт предоставляет HTTP endpoints для live HLS, archive HLS, MP4 export,
preview, статуса записи, WebRTC/WHEP и management API. Для каждой камеры можно
использовать embed-плеер, который встраивается в сторонний web-интерфейс.

Sesame DVR можно интегрировать с:

- клиентскими порталами;
- системами мониторинга объектов;
- внутренними CRM/ERP;
- биллинговыми системами;
- отраслевыми платформами для охраны, строительства, логистики, производства и
  удалённого контроля.

## Высокая производительность

Sesame DVR спроектирован как лёгкий и эффективный видеосервер. В архитектуре
используются решения, которые снижают лишнюю нагрузку и позволяют обслуживать
крупные инсталляции на подходящем серверном железе.

Ключевые элементы производительности:

- Elixir supervision tree для устойчивого управления множеством потоков;
- native `ffmpeg_nif` ingest path без отдельного OS-процесса `ffmpeg` на
  каждую совместимую камеру;
- packet copy/remux вместо тяжёлого транскодирования там, где это возможно;
- прямая запись self-initializing fMP4 segments в выбранный storage volume без
  staging-коммиттеров в hot path;
- файловый DVR-архив без обязательной базы данных в hot path записи;
- hour indexes и per-volume catalogs для быстрых рестартов и дешёвых status
  checks;
- event-driven обработка новых сегментов;
- отдача DVR-сегментов через nginx напрямую с диска при публичной публикации;
- параллельные bulk start/stop/enable/disable операции с контролируемым
  лимитом параллелизма;
- отдельный ONVIF runtime для крупных инсталляций, чтобы сбор событий не мешал
  основному видеосервису;
- ограничение фоновых задач и диагностических операций, чтобы они не забирали
  ресурсы у записи.

За счёт этого Sesame DVR хорошо подходит для сценариев, где нужно одновременно
вести запись, отдавать live-видео, обслуживать архивные запросы и сохранять
управляемую нагрузку на CPU, память и диск.

Фактическое количество камер зависит от сервера, bitrate, codec-профилей,
дисковой подсистемы, сети, retention и режима просмотра. При правильном подборе
железа продукт может использоваться как основа для крупных DVR/NVR-инсталляций
с большим количеством потоков.

## Надёжность и эксплуатация

Sesame DVR работает как `systemd` service и рассчитан на серверную эксплуатацию.
Сервис следит за состоянием потоков, ведёт runtime-логи, показывает диагностику
сервера и позволяет быстро понять, где проблема: камера, сеть, диск, CPU,
ingest или внешний доступ.

Архив хранится в понятной файловой структуре, а конфигурация и license state
находятся в стандартных системных директориях. Это упрощает резервное
копирование, обслуживание и диагностику.

## Быстрая установка и обновления

Protected-версия устанавливается одной командой через bootstrap installer.
Инсталлятор:

- скачивает готовый protected artifact;
- устанавливает системные зависимости;
- регистрирует `systemd` service;
- активирует лицензию через license server;
- может настроить nginx-публикацию;
- может выпустить Let's Encrypt сертификат для HTTPS;
- выводит ссылку для первого входа в web-интерфейс.

Обновление выполняется штатной командой:

```bash
sudo sesame-dvr-update
```

После обновления система сохраняет информацию о предыдущем release. Если новая
версия на конкретном сервере повела себя некорректно, администратор может
запустить штатный rollback на предыдущую установленную версию без ручной
перекладки файлов.

Клиенту не нужно собирать проект из исходников на своём сервере.

## Для каких клиентов подходит

Sesame DVR особенно полезен для:

- интеграторов видеонаблюдения;
- объектов с большим количеством IP-камер;
- строительных площадок и удалённых объектов;
- складов, офисов, производств и коммерческой недвижимости;
- сервисов, которым нужно встроить live/archive video в личный кабинет;
- компаний, которым нужен собственный контролируемый видеоархив;
- проектов, где важны API, web-доступ и гибкая публикация видео наружу.

## Почему выбирают Sesame DVR

- Быстрое развёртывание без локальной сборки.
- Live и архив в браузере.
- WebRTC, HLS, MP4 export и embed-плеер.
- Управление потоками и диагностикой из web UI.
- ONVIF events и timeline движения.
- SingleVolume/MultiVolume хранилище для больших архивов.
- Высокая производительность за счёт native ingest, packet copy и эффективного
  индекса архива.
- Лицензионный контроль камер, retention и активных playback client users.
- Прозрачное хранение данных на диске.
- Возможность публичной HTTPS-публикации через nginx.
- Лицензирование, обновления и штатный rollback через защищённый license server.

## Короткое позиционирование

Sesame DVR — это лёгкий, производительный и гибкий DVR/NVR-сервер для IP-камер,
который объединяет запись архива, live-просмотр, web-администрирование,
ONVIF-события, API-интеграции и публичную публикацию видео в одном решении.
